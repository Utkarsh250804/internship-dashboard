2. SQL Query Using INNER JOIN

SELECT 
Employees.EmployeeID,
Employees.EmployeeName,
Departments.DepartmentName,
Departments.DepartmentHead
FROM Employees
INNER JOIN Departments
ON Employees.DepartmentID = Departments.DepartmentID;


3. Explanation

The INNER JOIN combines records from both tables based on a matching value in the DepartmentID column.

Employees.DepartmentID matches Departments.DepartmentID to retrieve the department name and department head for each employee.


4. Role of Keys in Joining Tables

Primary Key:
DepartmentID in the Departments table acts as the Primary Key because it uniquely identifies each department.

Foreign Key:
DepartmentID in the Employees table acts as a Foreign Key because it references the DepartmentID in the Departments table.

Using these keys allows the database to correctly match employees with their respective departments.
